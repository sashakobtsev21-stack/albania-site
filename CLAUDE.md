# CLAUDE.md — проект Albania Guidebook

Контентный сайт-справочник по всей Албании **только на английском языке (en)** — единственный язык, на корне `/` (ru/uk удалены 2026-06-22): достопримечательности, города, еда (директория проверенных мест), маршруты, транспорт, аренда авто, страховки, развлечения, релокация, новости. Места отбираются по высокому рейтингу (>4★ из нескольких источников) и сверяются с опытными путешественниками, гидами и местными жителями. Диверсифицированная партнёрская монетизация (экскурсии, жильё, аренда, страховки, eSIM). Единственный источник правды по продукту, нише и приоритетам — **SPEC.md**. Этот файл — правила работы внутри репозитория. При конфликте приоритет: SPEC.md → CLAUDE.md → всё остальное.

## Стек и принципы
- **Astro 6 + Content Collections**, полная статика (SSG). Никаких SSR, БД, CMS.
- **Tailwind CSS 4**. Дизайн-токены — только из `src/styles/tokens.css` (SPEC §10). Хардкод hex-цветов в компонентах запрещён.
- **Без React.** Интерактив — нативный JS внутри Astro-компонентов, и только там, где без него нельзя: карта Leaflet, мобильное меню, переключатель языка.
- Хостинг: **Cloudflare Workers (Static Assets)**. Деплой = push в `main` (Workers Build: `npm run build` → `wrangler deploy`). Превью-деплои на ветках.
- Изображения: webp/avif через `astro:assets`, lazy-load, каждая ≤ 200 КБ.

## Железные правила
1. **Один язык — английский (en).** Сайт одноязычный: статья = одна **en**-версия (ru/uk удалены 2026-06-22). Контент лежит только в `src/content/<коллекция>/en/`. `check-parity` следит, чтобы контента вне `/en/` не было. Обновление фактов = правка en-версии.
2. **Партнёрские ссылки** — только через `/go/{partner}?c={slug}` с `rel="sponsored nofollow noopener"`. Прямые URL партнёрок в контенте запрещены. На статью 1–3 AffiliateBox, не больше.
3. **URL не меняются** после индексации. Ошибся в slug — 301 в `public/_redirects`, а не переименование.
4. **Факты не выдумывать.** Цены, адреса, телефоны, условия — только из брифа `briefs/{slug}.md`. Нет данных → оставить `<!-- TODO: факт от владельца -->` и сказать об этом, а не сочинять. Рейтинги/отзывы из внешних источников (Google, TripAdvisor) дословно не перепечатывать — это нарушает их ToS; использовать как критерий отбора и ссылку, добавляя собственное описание (SPEC §15).
5. **Перф-бюджет (мобайл):** LCP < 2.0 с, CLS < 0.1, JS на странице < 50 КБ. Любая новая зависимость тяжелее 10 КБ — спросить владельца.
6. **Доступность:** контраст ≥ 4.5:1, видимый focus, alt у каждого изображения, рабочая навигация с клавиатуры.
7. **SEO-обвязка каждой страницы:** title ≤ 60 зн., description ≤ 155, canonical, self-hreflang `en` + `x-default` → тот же URL (сайт одноязычный — других alternate нет), OG-картинка, schema.org по типу шаблона (SPEC §14).
8. **Сторонние скрипты:** только одна аналитика (SPEC §17). Никаких виджетов соцсетей, онлайн-чатов, тег-менеджеров.
9. **Доки не отстают от кода + commit/push (правило дисциплины — обязательно).** После КАЖДОЙ доработки синхронизировать документацию: `PROGRESS.md` (статус/роадмап + дата), `HANDOFF.md` (снимок: коммит, число страниц, «где остановились»), `AUDIT.md` (отметить закрытое/новое) **и все затронутые доки** (убрать старое, внести новое); при изменении продукта/правил — `SPEC.md` / этот `CLAUDE.md`; авто-память + её зеркало `docs/memory/`. **Затем сразу `git commit` + `git push` в `main` — без напоминаний.** **Доработка без актуализации доков и пуша НЕ считается завершённой.** Деталь: перед push — `git pull` (`fetch` + `rebase origin/main`), т.к. ежедневный авто-джоб новостей пушит в `main` первым.

## Контент-конвейер (SPEC §12)
`briefs/{slug}.md` (ключевой запрос, вопросы читателя, ФАКТЫ владельца, цены, фото) → **скелет:** `npm run new -- <тип> <slug> --title "…"` (генерит ОДНУ en-версию + папку фото + DoD-чеклист в комментарии) → черновик en по шаблону → фото (CC/свои) → **fact-check → content-editor** → коммит, draft снимается. Порядок именно такой: сначала факты сверяются (`fact-checker`), потом финальный приёмщик (`content-editor`).

## Definition of Done страницы
- `build` и `check` зелёные; Lighthouse mobile: Performance / SEO / Accessibility ≥ 90
- en-версия на месте (сайт одноязычный), self-hreflang en + x-default, schema.org проходит Rich Results Test
- дата обновления видима; AffiliateBox ведёт на рабочий /go/-редирект
- внутренние ссылки: минимум 2 на связанные статьи; хаб раздела ссылается на статью

### DoD по типам контента (gate `npm test` → check-photos + check-interlinks ≥2; полный список — в скелете `new-content.mjs`)
- **Статья/Новость:** cover (CC c `coverCredit`/своё) + ≥3 инлайн-figure ИЛИ ≥6 gallery — **фото обязательны**; факты только из брифа/источника (`sources`), иначе TODO; `visit` только с источником+`checkedAt`; ≥2 внутр. ссылки; title ≤60, desc ≤155; self-hreflang en + x-default; 1–3 AffiliateBox через `/go/`.
- **Маршрут:** cover обязателен; `days/distanceKm/budgetFrom/stops` заполнены; после `stops` — `node scripts/build-route-geometry.mjs`; далее как статья.
- **Место (ресторан):** отбор по консенсусу >4★ из нескольких источников + наш `ratingNote/review` своими словами (чужие отзывы дословно нельзя, §15); cover обязателен; `geo.coord`+`mapUrl`; `hours/dishes` только из источника; `verifiedAt`=дата.
- **Услуга:** `name/summary/rubric/city`; cover опц. (директория — текст-карточка); `url/contact` реальные (без `example.com`); `sponsored:true` только платное; `demo:true` только для честного примера (бейдж «Пример», noindex).
- **Город:** `name/region/coord/summary`; cover обязателен; связать с путеводителем (`article: reference`) при наличии.

## Стандарты публикации (объём и фото) — обязательны
**Объём текста:**
- **Статья:** **1200–2000 слов.** Первый абзац — прямой ответ за 2–3 предложения; H2/H3 под вопросы читателя; без «воды» и без FAQ-блока.
- **Новость:** **1500–2000 знаков** (~250–350 слов), **3–5 разделов**, **≥4 внутренних ссылки**.

**Минимум фото** (каждое уникально на материал, **webp ≤200 КБ**, своё или CC/PD — Wikimedia/Openverse/Unsplash — с атрибуцией «автор + лицензия»):
- **Статья:** **≥5** (cover + ≥4 инлайн/галерея).
- **Новость:** **≥2** (cover + ≥1 инлайн).
- **Маршрут:** cover + фото **каждой** остановки.
- **Город:** **≥10**; **столица (Тирана):** **≥15**.

**Модель «город = статья».** Город-путеводитель — это статья с `category: 'cities'`; отдельная коллекция `cities` для городов-путеводителей по этой модели НЕ используется.

**Тон.** Живой, человечный, как пишет носитель языка (uk — живая мова, не калька/суржик; en — как пишет носитель). **НЕ ИИ-генерёнка.** Факты — только из источника/брифа (цены/адреса/часы + «уточняйте» + дата проверки; нет данных → `<!-- TODO -->` + источник). Даты/время/числа — только из источника **текущего года**. Валюта — **албанский лек (ALL)**.

**Партнёрские ссылки** — только через `/go/{partner}` с `rel="sponsored nofollow noopener"` (правило 2).

## Обновления формата (2026-06, согласовано с владельцем)
- **FAQ убран со всего сайта.** Нет FAQBlock и FAQPage-schema (пункт DoD про FAQPage больше не применять). Не возвращать без явного решения владельца; новую инфо вплетать в тело статьи.
- **Фото — инлайн в тексте + мини-галерея внизу.** Ключевые фото = raw `<figure class="figure">` в .md с подписью «что на фото» + CC-кредит (width/height → 0 CLS, lazy); остальные — во frontmatter `gallery`. Каждое фото один раз. Цель: Тирана ~20, города ~12–15 «по качеству».
- **Цена и часы — поле `visit`** (`price`/`hours`/`note`/`checkedAt`) → VisitInfo. Цены/часы не выдумывать (правило 4): только надёжный источник + дата проверки + «уточняйте»; нет данных → не указывать.
- **Порядок в хабах — `featuredOrder`** (меньше = выше). Флаг Албании в шапке — `/images/al-flag.svg` (официальный двуглавый орёл, **сделано**); фавикон/PWA-иконки — тот же орёл.
- **Язык:** сайт **одноязычный — только английский (en)** на корне `/` (ru/uk полностью удалены 2026-06-22: страницы, контент, словари, LangSwitcher; `/ru/*` и `/uk/*` → 301 на корень в `public/_redirects`). Локальный албанский (`sq`) НЕ делаем. Новый контент — только en.
- **Лучшие фото + развёрнутое описание (правило владельца 2026-06-16).** При добавлении ЛЮБОЙ статьи/раздела — только самые качественные фото (CC/свои, отбор глазами; слабые/мусорные кадры не публиковать) и развёрнутое описание ПО АНАЛОГИИ с текущими статьями (тот же объём и структура: инлайн-фигуры с подписями + мини-галерея, выверенные факты с источником, «как доехать», visit). Не тонкие «заглушки на отвяжись».
- **Кросс-сайтовая перелинковка (хаб `docs/STANDARDS.md` §«Перелинковка»).** На уместную тему можно ссылаться на сестринский сайт сети обычной markdown-ссылкой на полный `https://`-URL — контекстно и только когда цель реально резолвится (200). Движок (`rehypeExternalLinks`) сам навешивает `rel="nofollow noopener noreferrer"` → нулевой PBN-риск, без футер-ферм и взаимных «ссылочных бирж». (Внутренняя перелинковка — в DoD выше, не дублируем.)
- **Синхронизация доков после каждой доработки (правило владельца 2026-06-16).** Завершив доработку — обновить `PROGRESS.md` (роадмап/статус + дата) и `HANDOFF.md` (снимок: коммит, число страниц, «где остановились»); при изменении продукта/правил — `SPEC.md`/`CLAUDE.md`; `AUDIT.md` — отметить закрытое; авто-память + её зеркало `docs/memory/`. Доки не должны отставать от кода.

- Не добавлять React, CMS, базы данных и платные сервисы без явного решения владельца.
- Не клонировать инструменты (my_agents, ui-ux-pro-max) внутрь этого репозитория — они подключаются как плагины Claude Code, см. SETUP-GUIDE.md.
- Не генерировать массово страницы без уникальной фактуры: «SEO-генерёнка» = риск бана и прямое нарушение SPEC §15.
- Не трогать логику `/go/` без обновления таблицы партнёрок в SPEC §16.
